# Design audit

Generated by `npm run audit:scaffold` from PNGs in
`design-audit/screenshots/`. One section per captured slug, plus any
manual GAP entries documented up-front.

Each section has:
- **Path** — where the screenshot was taken
- **Desktop / Mobile** — links to the captured PNGs (relative to this file)
- **Status** — TODO until you triage. Suggested values: OK, NEEDS-WORK, BROKEN, GAP
- **Notes** — what to fix, or why it's correct

This file is **idempotent**: re-running `npm run audit:scaffold` after a
fresh `npm run audit:capture` will only ever **append** sections for newly
captured slugs. Existing sections (and your filled-in notes) are preserved
verbatim.

## reset-password

- **Path:** /reset-password
- **Desktop:** _GAP — not auto-captured_
- **Mobile:** _GAP — not auto-captured_
- **Status:** GAP
- **Notes:** Reaching the reset-password form requires a Supabase recovery
  session, which is created only by clicking the link in the
  password-reset email. The capture script cannot reproduce this without
  intercepting mail. To audit this page manually: trigger `/login` →
  "Forgot password?" → click the email link, screenshot the resulting
  page, and drop the files at
  `design-audit/screenshots/desktop/reset-password.png` (and the mobile
  equivalent). Once those exist on disk this section will be skipped on
  the next scaffold run, and you can re-fill the Status / Notes fields by
  hand.
